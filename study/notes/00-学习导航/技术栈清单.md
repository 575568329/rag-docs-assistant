## 🗄️ 数据存储方案

### 向量存储
- **FileStore（本地）**：data/vectors.json
  - ✅ 优势：无依赖、开发快速、支持混合搜索
  - ❌ 劣势：不支持并发、性能有限
  
- **ChromaDB（可选）**：需独立服务
  - ✅ 优势：高性能、支持大规模数据
  - ❌ 劣势：需额外部署、不支持关键词搜索

### 元数据存储
- **本地 JSON**：data/db.json
  - 知识库、文档、对话、收藏记录

### 图谱存储
- **本地 JSON**：data/graph.json
  - Graphology 序列化格式

## 🔍 核心算法与模式

### 1. 向量检索算法
```
余弦相似度 (Cosine Similarity)
公式: similarity = dot(a, b) / (norm(a) * norm(b))
范围: [-1, 1]，1 表示完全相同
```

### 2. 混合搜索（RRF 融合）
```
RRF (Reciprocal Rank Fusion)
公式: score = Σ(1 / (k + rank_i))
k = 60（常数）
融合向量搜索和关键词搜索的排序结果
```

### 3. 文本切片策略
```
1. 按 Markdown 标题分段（# ~ ######）
2. 大段落按固定大小切分（1000 字符）
3. 重叠机制（100 字符 overlap）
```

### 4. 实体抽取
```
使用 LLM + Zod Schema
实体类型: Person, Organization, Location, Concept, Event, Document
关系类型: BELONGS_TO, RELATED_TO, LOCATED_AT, PART_OF, CAUSED_BY
```

## 📊 性能参数

| 参数 | 值 | 说明 |
|------|---|------|
| **相似度阈值** | 0.45 | 低于此值的结果被过滤 |
| **Top-K** | 5 | 返回前 K 条检索结果 |
| **多轮对话上下文** | 最近 5 条消息 | 用于拼接搜索查询 |
| **搜索查询截断** | 500 字符 | 避免稀释 embedding |
| **切片大小** | 1000 字符 | 固定大小切片 |
| **切片重叠** | 100 字符 | 保持上下文连续性 |
| **文档大小限制** | 10MB | 单文件上传限制 |
