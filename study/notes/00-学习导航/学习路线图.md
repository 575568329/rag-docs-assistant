## 🎯 各领域知识点拆解

### A. Next.js 基础 (15%)
- [ ] App Router 路由组织（`(platform)` 分组路由）
- [ ] API Routes 写法（`route.ts` 的 GET/POST 导出）
- [ ] 动态路由参数（`[id]`、`[docId]`）
- [ ] Server Components vs Client Components
- [ ] 流式响应（Streaming Response / SSE）
- [ ] `next.config.ts` 配置
- [ ] 布局嵌套（`layout.tsx`）

> ⚠️ **注意**：本项目使用的 Next.js 16 有破坏性变更，写代码前先读 `node_modules/next/dist/docs/`

### B. 向量检索 (25%) ⭐ 核心
- [ ] 文本向量化（Embedding）原理
- [ ] 调用智谱 embedding-3 API
- [ ] 余弦相似度计算
- [ ] 向量存储设计（FileStore / ChromaDB 抽象）
- [ ] **混合搜索**：向量 + 关键词
- [ ] **RRF（Reciprocal Rank Fusion）融合算法**
- [ ] 相似度阈值过滤（0.45）
- [ ] Top-K 检索
- [ ] 多知识库跨库检索

### C. LangChain 集成 (20%) ⭐ 核心
- [ ] AI SDK v6 核心 API（`streamText`、`generateObject`）
- [ ] 流式输出（streaming）
- [ ] UIMessage 消息格式转换
- [ ] 多轮对话上下文管理
- [ ] Prompt 工程（系统提示词构建）
- [ ] 结构化输出（Zod schema）
- [ ] 消息元数据传递（sources）
- [ ] `onFinish` 回调持久化

### D. 文档处理 (15%)
- [ ] 多格式解析（PDF / DOCX / XLSX / TXT / MD）
- [ ] pdfjs-dist / mammoth / exceljs 用法
- [ ] **文本切片策略**：按标题分段 + 固定大小
- [ ] 切片重叠（overlap）机制
- [ ] 元数据追溯（filename / heading / chunkIndex）

### E. 知识图谱 (15%)
- [ ] 实体抽取（LLM + Zod schema）
- [ ] 实体类型设计（Person/Org/Location/Concept/Event/Document）
- [ ] 关系类型设计（BELONGS_TO/RELATED_TO/...）
- [ ] Graphology 图库使用
- [ ] 图谱序列化与查询
- [ ] 邻域查询（N 层）
- [ ] react-force-graph-2d 力导图可视化

### F. 工程化实践 (10%)
- [ ] 结构化日志（logger）
- [ ] 错误处理与优雅降级
- [ ] 本地 JSON 持久化设计
- [ ] 状态管理
- [ ] 性能优化（批量处理、避免 N+1）

## 📈 学习节奏建议

- **每天 1 个里程碑**（核心模块 M3/M4 可拆 2 天）
- **每个模块必走**：理论笔记 → 对照参考 → 从零实现 → 测试验证 → 会话记录
- **遇到卡点**：先记录到易错点，再对比 `master` 分支找答案
- **总周期**：约 9-10 天，35-40 小时
