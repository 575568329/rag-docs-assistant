# 🔄 数据流全链路图

> 从用户操作到系统响应的完整数据流，帮助理解各模块协作关系

---

## 📤 流程 1：文档上传与摄入

```
用户上传文档
    │
    ├─> 1. 前端：文件拖拽/选择（DataPage.tsx）
    │      └─> FormData 构建（文件 + kbId）
    │
    ├─> 2. API：POST /api/kb/[id]/upload
    │      │
    │      ├─> 2.1 校验知识库存在性（db.listKB）
    │      ├─> 2.2 校验文件（大小、格式、同名）
    │      ├─> 2.3 保存原始文件到 data/uploads/
    │      │
    │      ├─> 2.4 文档解析（doc-parser.ts）
    │      │      ├─ .pdf   → pdfjs-dist 提取文本
    │      │      ├─ .docx  → mammoth 解析
    │      │      ├─ .xlsx  → exceljs 逐行读取
    │      │      └─ .txt/.md → 直接读取
    │      │
    │      ├─> 2.5 文本切片（chunker.ts）
    │      │      ├─ 按 Markdown 标题分段
    │      │      ├─ 大段落按 1000 字符切分
    │      │      ├─ 重叠 100 字符
    │      │      └─ 附加元数据：{filename, heading, chunkIndex}
    │      │
    │      ├─> 2.6 批量向量化（embedding.ts）
    │      │      └─> 调用智谱 embedding-3 API
    │      │             POST https://open.bigmodel.cn/api/paas/v4/embeddings
    │      │             返回：float[] (维度 1024 或 2048)
    │      │
    │      ├─> 2.7 存入向量库（vector-store/）
    │      │      ├─ FileStore → data/vectors.json
    │      │      │     {
    │      │      │       "kb-{kbId}": {
    │      │      │         ids: [...],
    │      │      │         texts: [...],
    │      │      │         vectors: [...],
    │      │      │         metas: [...]
    │      │      │       }
    │      │      │     }
    │      │      └─ ChromaDB → HTTP POST /api/v1/collections/{name}/add
    │      │
    │      ├─> 2.8 实体抽取（entity-extractor.ts）
    │      │      └─> AI SDK generateObject + Zod schema
    │      │             提取：Person, Org, Location, Concept, Event, Document
    │      │             提取关系：BELONGS_TO, RELATED_TO, ...
    │      │
    │      ├─> 2.9 构建知识图谱（graph-store.ts）
    │      │      └─> Graphology: addNode, addEdge
    │      │             持久化到 data/graph.json
    │      │
    │      └─> 2.10 记录元数据（db.ts）
    │             └─> db.addDoc(kbId, filename, chunkCount, chunkIds)
    │                    持久化到 data/db.json
    │
    └─> 3. 响应：200 OK
           └─> 前端刷新文档列表
```

**关键文件路径**：
- 入口：`src/app/api/kb/[id]/upload/route.ts`
- 解析：`src/lib/doc-parser.ts`
- 切片：`src/lib/chunker.ts`
- 向量化：`src/lib/embedding.ts`
- 向量存储：`src/lib/vector-store/file-store.ts`
- 实体抽取：`src/lib/entity-extractor.ts`
- 图谱存储：`src/lib/graph-store.ts`

## 💬 流程 2：RAG 对话问答

```
用户提问
    │
    ├─> 1. 前端：聊天输入（ChatPage.tsx）
    │      └─> useChat Hook（AI SDK）
    │             POST /api/chat
    │             body: { messages, kbId, conversationId }
    │
    ├─> 2. API：POST /api/chat
    │      │
    │      ├─> 2.1 提取用户问题
    │      │      userQuery = messages[messages.length - 1].content
    │      │
    │      ├─> 2.2 多轮对话优化（buildSearchQuery）
    │      │      ├─ 取最近 5 条消息 + 当前问题
    │      │      ├─ 拼接为完整上下文
    │      │      └─ 截断到 500 字符（避免稀释 embedding）
    │      │
    │      ├─> 2.3 问题向量化
    │      │      queryVector = await getEmbedding([searchQuery])
    │      │
    │      ├─> 2.4 解析检索范围（resolveKnowledgeBaseScope）
    │      │      ├─ kbId 有值 → ["kb-{kbId}"]
    │      │      └─ kbId 为空 → ["kb-0", "kb-1", ...] (所有知识库)
    │      │
    │      ├─> 2.5 混合检索（vectorStore.search）
    │      │      │
    │      │      ├─ FileStore 混合搜索：
    │      │      │   ├─ 2.5.1 向量搜索（余弦相似度）
    │      │      │   │     similarity = dot(queryVec, docVec) / (norm(q) * norm(d))
    │      │      │   │     排序得到 vectorRank[]
    │      │      │   │
    │      │      │   ├─ 2.5.2 关键词搜索（TF-IDF）
    │      │      │   │     分词 → 计算 TF-IDF → 排序得到 keywordRank[]
    │      │      │   │
    │      │      │   └─ 2.5.3 RRF 融合
    │      │      │         score = Σ(1 / (60 + rank_i))
    │      │      │         返回 Top-5
    │      │      │
    │      │      └─ ChromaDB 纯向量搜索：
    │      │            POST /api/v1/collections/{name}/query
    │      │            过滤 similarity < 0.45
    │      │
    │      ├─> 2.6 构建上下文（buildContext）
    │      │      ├─ 拼接检索到的文档内容
    │      │      ├─ 标注来源编号 [1], [2], ...
    │      │      └─ 构建 SourceRef[] {index, kbId, docId, filename, heading, score}
    │      │
    │      ├─> 2.7 构建系统提示词（buildSystemPrompt）
    │      │      ├─ 有相关内容：要求引用 [编号]
    │      │      └─ 无相关内容：标注"AI 补充回答"
    │      │
    │      ├─> 2.8 调用 LLM 流式生成
    │      │      AI SDK streamText({
    │      │        model: glm-4-flash,
    │      │        system: systemPrompt,
    │      │        messages: [...history, userQuery]
    │      │      })
    │      │      └─> SSE 流式返回
    │      │
    │      ├─> 2.9 onFinish 持久化
    │      │      ├─ db.addConversationMessage(conversationId, 'user', userQuery)
    │      │      └─ db.addConversationMessage(conversationId, 'assistant', text, {sources})
    │      │
    │      └─> 2.10 返回流式响应
    │             └─> messageMetadata 携带 sources 信息
    │
    └─> 3. 前端：流式显示（ChatPage.tsx）
           ├─> Markdown 渲染回答
           └─> 显示来源卡片（SourceCard.tsx）
```

**关键文件路径**：
- 入口：`src/app/api/chat/route.ts`
- 向量化：`src/lib/embedding.ts`
- 混合搜索：`src/lib/vector-store/file-store.ts:120-180`
- 前端组件：`src/components/chat/ChatPage.tsx`

## 🕸️ 流程 3：知识图谱查询

```
用户查询图谱
    │
    ├─> 1. 前端：图谱页面（GraphPage.tsx）
    │      │
    │      ├─> 场景 1：全图加载
    │      │      GET /api/graph?kbId={kbId}
    │      │
    │      └─> 场景 2：搜索节点
    │             GET /api/graph?query={keyword}&kbId={kbId}
    │
    ├─> 2. API：GET /api/graph
    │      │
    │      ├─> 2.1 加载图谱数据（graph-store.ts）
    │      │      从 data/graph.json 反序列化为 Graphology 实例
    │      │
    │      ├─> 2.2 按 kbId 过滤
    │      │      ├─ kbId 有值 → 过滤 node.attributes.kbId === kbId
    │      │      └─ kbId 为空 → 返回全部节点
    │      │
    │      ├─> 2.3 搜索节点（如有 query）
    │      │      graph.forEachNode((node, attrs) => {
    │      │        if (attrs.label.includes(query)) matches.push(node)
    │      │      })
    │      │
    │      ├─> 2.4 获取邻域（如有 nodeId）
    │      │      ├─ 1 层邻居：graph.neighbors(nodeId)
    │      │      └─ N 层递归：BFS 遍历
    │      │
    │      └─> 2.5 返回图数据
    │             {
    │               nodes: [{id, label, type, ...}, ...],
    │               edges: [{source, target, relationType, ...}, ...]
    │             }
    │
    └─> 3. 前端：力导图渲染（GraphCanvas.tsx）
           └─> react-force-graph-2d
                 ├─ 节点点击 → 显示详情面板（NodeDetail.tsx）
                 ├─ 节点收藏 → POST /api/graph/favorites
                 └─ 节点搜索 → GraphSearch.tsx
```

**关键文件路径**：
- 入口：`src/app/api/graph/route.ts`
- 图谱存储：`src/lib/graph-store.ts`
- 前端可视化：`src/components/graph/GraphCanvas.tsx`

## 📊 数据持久化结构

### data/db.json
```json
{
  "knowledgeBases": [
    { "id": "know_0", "name": "默认知识库", "createdAt": "..." }
  ],
  "documents": [
    {
      "id": "doc_123",
      "kbId": "know_0",
      "filename": "example.pdf",
      "chunkCount": 15,
      "chunkIds": ["chunk_1", "chunk_2", ...],
      "uploadedAt": "..."
    }
  ],
  "conversations": [
    { "id": "conv_456", "kbId": "know_0", "title": "...", "createdAt": "..." }
  ],
  "conversationMessages": [
    {
      "id": "msg_789",
      "conversationId": "conv_456",
      "role": "user|assistant",
      "content": "...",
      "metadata": { "sources": [...] },
      "timestamp": "..."
    }
  ],
  "favorites": [
    { "id": "fav_1", "kbId": "know_0", "nodeId": "node_abc", "createdAt": "..." }
  ]
}
```

### data/vectors.json（FileStore）
```json
{
  "kb-know_0": {
    "ids": ["chunk_1", "chunk_2", ...],
    "texts": ["文本内容1", "文本内容2", ...],
    "vectors": [[0.1, 0.2, ...], [0.3, 0.4, ...], ...],
    "metas": [
      { "filename": "example.pdf", "heading": "第一章", "chunkIndex": 0 },
      ...
    ]
  }
}
```

### data/graph.json（Graphology）
```json
{
  "nodes": [
    {
      "key": "node_abc",
      "attributes": {
        "label": "人工智能",
        "type": "Concept",

        "docId": "doc_123"
      }
    }
  ],
  "edges": [
    {
      "source": "node_abc",
      "target": "node_def",
      "attributes": {
        "relationType": "RELATED_TO"
      }
    }
  ]
}
```
