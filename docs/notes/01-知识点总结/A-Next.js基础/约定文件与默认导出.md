## 📝 定义

**约定文件（Convention Files）**: Next.js App Router 通过**文件名约定**来识别框架级别的特殊文件（如布局、页面、加载态等）。这些文件**必须使用 `export default` 导出 React 组件**，否则框架无法识别，会直接报错。

> 核心一句话：在 `app/` 目录下，文件名是约定的（如 `page.tsx`、`layout.tsx`），框架只读 default export，且必须是 React 组件。

---

## 🎯 约定文件清单

| 文件名 | 作用 | 是否必须默认导出 | 必须是 React 组件 | 备注 |
|--------|------|----------------|-----------------|------|
| `page.tsx` | 页面入口 | ✅ 是 | ✅ 是 | 决定路由 URL 是否可访问 |
| `layout.tsx` | 布局（持久化） | ✅ 是 | ✅ 是 | 接收 `children` props |
| `loading.tsx` | 加载态 UI | ✅ 是 | ✅ 是 | 基于 React Suspense |
| `error.tsx` | 错误边界 UI | ✅ 是 | ✅ 是 | 必须是 Client Component |
| `not-found.tsx` | 404 UI | ✅ 是 | ✅ 是 | 配合 `notFound()` 触发 |
| `template.tsx` | 模板（每次重新渲染） | ✅ 是 | ✅ 是 | 类似 layout 但不持久化 |
| `route.ts` | API 路由 | ❌ **例外** | ❌ 否 | 用命名导出 `GET`/`POST`/... |
| `default.tsx` | 并行路由兜底 | ✅ 是 | ✅ 是 | 平行路由（@slot）配套 |

## 💻 错误代码 vs 正确代码

### ❌ 错误 1：空文件

```tsx
// app/(platform)/layout.tsx —— 空文件
```

**报错**:
```
Error: The default export is not a React Component in "/(platform)/layout"
```

**原因**: 框架找不到 default export，更别说判断它是不是组件了。

---

### ❌ 错误 2：只有命名导出

```tsx
// app/(platform)/layout.tsx
export function PlatformLayout({ children }: { children: React.ReactNode }) {
  return <div>{children}</div>;
}
```

**报错**:
```
Error: The default export is not a React Component in "/(platform)/layout"
```

**原因**: 框架只读 `export default`，命名导出对 Next.js 约定文件不生效。

---

### ❌ 错误 3：默认导出了非组件

```tsx
// app/page.tsx
export default {
  name: 'HomePage',
  description: '首页',
}; // ❌ 默认导出的是普通对象，不是函数组件
```

**报错**:
```
Error: The default export is not a React Component in "/page"
```

**原因**: default export 存在，但不是 React 组件（没有返回 JSX 的函数）。

---

### ❌ 错误 4：默认导出了字符串/数字

```tsx
// app/loading.tsx
export default 'Loading...'; // ❌ 字符串不是组件
```

**报错**: 同上 `The default export is not a React Component`。

---

### ✅ 正确写法（任选一种）

```tsx
// 写法 1：函数声明 + 默认导出（推荐）
export default function PlatformLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return <div>{children}</div>;
}
```

```tsx
// 写法 2：先声明再默认导出
function PlatformLayout({ children }: { children: React.ReactNode }) {
  return <div>{children}</div>;
}
export default PlatformLayout;
```

```tsx
// 写法 3：箭头函数 + 默认导出
const PlatformLayout = ({ children }: { children: React.ReactNode }) => {
  return <div>{children}</div>;
};
export default PlatformLayout;
```

---

### ✅ 例外：`route.ts` 用命名导出

```ts
// app/api/health/route.ts
import { NextResponse } from 'next/server';

// ✅ 命名导出，方法名即 HTTP 动词
export async function GET() {
  return NextResponse.json({ status: 'ok' });
}

export async function POST(request: Request) {
  const body = await request.json();
  return NextResponse.json({ received: body });
}
```

`route.ts` 是**唯一例外**：用命名导出 `GET` / `POST` / `PUT` / `DELETE` / `PATCH` / `HEAD` / `OPTIONS`。

## 🔍 错误的本质

Next.js 内部对约定文件的处理逻辑大致如下（伪代码）：

```ts
// 伪代码：Next.js 内部加载约定文件
async function loadConventionFile(filePath: string) {
  const mod = await import(filePath);
  const Component = mod.default;

  // 1. 没有 default export → undefined
  // 2. default export 不是函数 → 不是组件
  // 3. default export 是函数但不返回 JSX → 运行时才报
  if (typeof Component !== 'function') {
    throw new Error(
      `The default export is not a React Component in "${routePath}"`
    );
  }
  return Component;
}
```

**关键点**:
1. 框架**只读 `mod.default`**，命名导出会被忽略。
2. 类型检查只看是不是 `function`（含箭头函数、类组件）。
3. 空文件 → `mod.default === undefined` → 立即报错。

## ⚠️ 对比：普通文件 vs 约定文件

| 维度 | 普通组件文件（如 `components/Button.tsx`） | 约定文件（如 `app/page.tsx`） |
|------|---------------------------------------|-----------------------------|
| 位置 | 任意目录（一般在 `components/`、`lib/`） | 必须在 `app/` 下，按约定命名 |
| 导出方式 | 命名导出 / 默认导出**都可以** | **必须 `export default`**（route.ts 除外） |
| 导出内容 | 任意（组件、工具函数、常量、类型） | **必须是 React 组件** |
| 引用方 | 由开发者手动 import 使用 | 由 Next.js 框架自动加载 |
| 文件名约束 | 无（推荐 PascalCase） | 严格约定（`page` / `layout` / `loading`...） |
| 命名导出示例 | `export const Button = ...` ✅ | `export const Page = ...` ❌ |
| 多个导出 | ✅ 可以（一个文件多个工具函数） | ⚠️ 可以有，但框架只用 default |
