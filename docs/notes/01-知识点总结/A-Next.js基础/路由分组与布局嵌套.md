## 📝 定义

**路由分组（Route Groups）**: Next.js App Router 提供的一种组织路由的语法，用括号 `(folder)` 包裹文件夹名，括号内的名称**不会出现**在 URL 路径中。

---

## 🎯 核心特点

| 特性 | 说明 |
|------|------|
| **语法** | 用括号包裹：`(name)` |
| **URL** | 括号内的名称不出现在 URL 中 |
| **作用** | 组织代码、共享布局、分组路由 |
| **命名** | 任意语义化名称（推荐 `platform`、`dashboard` 等） |

---

## 💡 为什么需要路由分组？

### 场景：3 个页面共享布局

需求：
- 对话页 `/chat`
- 数据页 `/data`
- 知识图谱页 `/graph`

这 3 个页面都需要：
- ✅ 相同的顶部导航
- ✅ 相同的左侧边栏
- ✅ 相同的布局结构

---

## 🔄 两种方案对比

### ❌ 方案 1：不使用路由分组
```
app/
├── layout.tsx          # 根布局
├── chat/
│   ├── layout.tsx      # 重复 1
│   └── page.tsx
├── data/
│   ├── layout.tsx      # 重复 2
│   └── page.tsx
└── graph/
    ├── layout.tsx      # 重复 3
    └── page.tsx
```

**问题**: 3 个 layout.tsx 重复代码，维护困难

---

### ✅ 方案 2：使用路由分组
```
app/
├── layout.tsx          # 根布局
└── (platform)/         # 路由分组（不影响 URL）
    ├── layout.tsx      # 平台共享布局（只写一次）
    ├── chat/page.tsx
    ├── data/page.tsx
    └── graph/page.tsx
```

**优势**:
- 只需要一个 layout.tsx
- 代码组织清晰
- URL 路径不变

---

## 📊 URL 对比

| 文件路径 | 实际 URL | 说明 |
|---------|---------|------|
| `app/(platform)/chat/page.tsx` | `/chat` | 括号被忽略 |
| `app/(platform)/data/page.tsx` | `/data` | 括号被忽略 |
| `app/(platform)/graph/page.tsx` | `/graph` | 括号被忽略 |
| `app/platform/chat/page.tsx` | `/platform/chat` ⚠️ | 不带括号则保留路径 |

## 🎨 布局嵌套渲染

### 访问 `/chat` 时的渲染流程

```
app/layout.tsx（根布局）
  └─ app/(platform)/layout.tsx（平台布局）
       └─ app/(platform)/chat/page.tsx（对话页面）
```

### 最终 HTML 结构
```html
<html>  <!-- 根布局 -->
  <body>
    <div class="h-screen flex flex-col">  <!-- 平台布局 -->
      <header>顶部导航</header>
      <div class="flex">
        <aside>左侧边栏</aside>
        <main>
          <h1>对话工作台</h1>  <!-- 页面内容 -->
        </main>
      </div>
    </div>
  </body>
</html>
```

## 💻 代码示例

### 平台布局 `app/(platform)/layout.tsx`

```tsx
'use client';  // ⚠️ 使用 usePathname 必须加这个

import { usePathname } from 'next/navigation';
import Link from 'next/link';  // ⚠️ 注意：路径全小写

export default function PlatformLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  const pathname = usePathname();

  const tabs = [
    { name: '对话', path: '/chat' },
    { name: '数据', path: '/data' },
    { name: '图谱', path: '/graph' },
  ];

  return (
    <div className="h-screen flex flex-col">
      <header className="h-16 border-b bg-white flex items-center px-6">
        <nav className="flex gap-6">
          {tabs.map((tab) => (
            <Link
              key={tab.path}
              href={tab.path}
              className={`px-4 py-2 rounded ${
                pathname === tab.path
                  ? 'bg-blue-500 text-white'
                  : 'text-gray-600 hover:bg-gray-100'
              }`}
            >
              {tab.name}
            </Link>
          ))}
        </nav>
      </header>
      <div className="flex-1 flex overflow-hidden">
        <aside className="w-64 bg-gray-50 border-r p-4">侧边栏</aside>
        <main className="flex-1 overflow-auto p-6">{children}</main>
      </div>
    </div>
  );
}
```

## 🔍 深度讲解

### 1. 为什么 layout.tsx 必须 `export default`？

Next.js 通过**约定（Convention）**来识别布局和页面：
- `layout.tsx` 的默认导出 = 布局组件
- `page.tsx` 的默认导出 = 页面组件

如果不是默认导出，Next.js 无法识别。

### 2. `children` 是什么？

`children` 是 React 的特殊 prop，表示**嵌套的子内容**。

```tsx
// 当访问 /chat 时
<PlatformLayout>
  <ChatPage />   {/* 这就是 children */}
</PlatformLayout>
```

### 3. 为什么要 `'use client'`？

`usePathname` 是 React Hook，只能在**客户端组件**中使用。

Next.js 默认所有组件是 Server Components（服务端组件），需要客户端能力时必须加 `'use client'` 标记。

## 🎯 路由分组的其他用途

### 1. 按功能分组
```
app/
├── (marketing)/      # 营销页面组
│   ├── layout.tsx
│   ├── about/
│   └── pricing/
└── (shop)/          # 商城页面组
    ├── layout.tsx
    ├── products/
    └── cart/
```

### 2. 按权限分组
```
app/
├── (public)/        # 公开页面
│   └── login/
└── (protected)/     # 需要登录
    ├── dashboard/
    └── settings/
```

## ⚠️ 常见错误

### 错误 1: 模块路径大小写错误
```tsx
// ❌ 错误
import Link from 'next/Link';

// ✅ 正确
import Link from 'next/link';
```

### 错误 2: 忘记 `'use client'`
```tsx
// ❌ 错误：Server Component 中使用 Hook
import { usePathname } from 'next/navigation';
export default function Layout({ children }) { ... }

// ✅ 正确：标记为 Client Component
'use client';
import { usePathname } from 'next/navigation';
export default function Layout({ children }) { ... }
```

### 错误 3: 命名导出而非默认导出
```tsx
// ❌ 错误
export function PlatformLayout({ children }) { ... }

// ✅ 正确
export default function PlatformLayout({ children }) { ... }
```

## 📚 类比理解

把布局想象成一个**相框**：

```tsx
function 相框({ 照片 }: { 照片: React.ReactNode }) {
  return (
    <div className="相框边框">
      {照片}  {/* 这里放入不同的照片 */}
    </div>
  );
}

// 使用时
<相框>
  <对话页 />  {/* 照片 = 对话页 */}
</相框>
```

不同的页面（chat/data/graph）就像不同的照片，放进同一个相框（layout）里。
