## 易错点列表

### ❌ 易错点 1: layout.tsx 默认导出错误

**优先级**: ⭐⭐⭐⭐

**涉及领域**: A. Next.js 基础

**错误现象**:
```
Error: The default export is not a React Component in "/chat/layout"
```

**错误理解**: 以为 `layout.tsx` 可以用任何方式导出组件

**正确理解**: Next.js 约定 `layout.tsx` 和 `page.tsx` **必须有一个默认导出**（`export default`），且必须是 React 组件

**错误代码**:
```tsx
// ❌ 错误 1：命名导出，没有 default
export function PlatformLayout({ children }) {
  return <div>{children}</div>;
}

// ❌ 错误 2：const 导出
export const PlatformLayout = ({ children }) => {
  return <div>{children}</div>;
};

// ❌ 错误 3：忘记导出
function PlatformLayout({ children }) {
  return <div>{children}</div>;
}
```

**正确代码**:
```tsx
// ✅ 正确：默认导出 + 函数声明
export default function PlatformLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return <div>{children}</div>;
}
```

**原因分析**: Next.js 框架通过约定（Convention）识别页面和布局，约定的方式就是默认导出

**后果**: 页面无法渲染，报错 "The default export is not a React Component"

**记忆点**: `layout.tsx` 和 `page.tsx` 必须 `export default function`

### ❌ 易错点 2: Next.js 模块路径大小写错误

**优先级**: ⭐⭐⭐⭐⭐

**涉及领域**: A. Next.js 基础

**错误现象**:
```
Module not found: Can't resolve 'next/Link'
```

**错误理解**: 以为 Windows 系统不区分大小写，写错也没关系

**正确理解**: Next.js 模块路径**严格大小写敏感**，必须全小写

**错误代码**:
```tsx
// ❌ 错误：路径大写
import Link from 'next/Link';
import Image from 'next/Image';
import { useRouter } from 'next/Navigation';
import { redirect } from 'Next/navigation';
```

**正确代码**:
```tsx
// ✅ 正确：路径全小写
import Link from 'next/link';
import Image from 'next/image';
import { useRouter, usePathname } from 'next/navigation';
import { redirect } from 'next/navigation';
```

**原因分析**:
- Linux/Mac 文件系统大小写敏感
- Node.js 模块解析严格区分大小写
- 即使在 Windows 上能跑，部署到 Linux 服务器会报错

**后果**: 模块无法解析，编译失败

**记忆点**: **包名小写，路径小写，组件名首字母大写**
```tsx
import Link from 'next/link';
//     ^^^^      ^^^^^^^^^
//     大写       小写
```

## 📚 知识点积累

### Q1: 路由分组 `(platform)` 是什么写法？

**回答**: Next.js 的**路由分组**（Route Groups）语法。

**特点**:
- 用括号包裹的文件夹名称**不会出现在 URL 路径中**
- 用于组织代码结构、共享布局
- 名称可以任意（如 `(platform)`、`(workspace)`、`(main)`）

**示例**:
| 文件路径 | 实际 URL |
|---------|---------|
| `app/(platform)/chat/page.tsx` | `/chat` |
| `app/(workspace)/chat/page.tsx` | `/chat` |

---

### Q2: 为什么用这种写法 `{ children }: { children: React.ReactNode }`？

**回答**: 这是 React + TypeScript 的标准组件写法，包含 3 个知识点：

1. **`export default`**: Next.js 要求布局必须默认导出
2. **`{ children }`**: 解构 props，直接取出 children
3. **`: { children: React.ReactNode }`**: TypeScript 类型注解

**完整拆解**:
```tsx
function PlatformLayout(
  { children }                       // 解构参数
  :                                  // 类型分隔符
  { children: React.ReactNode }      // 类型注解
) { }
```

**`React.ReactNode` 是什么**:
表示任何可以被 React 渲染的内容（字符串、数字、组件、数组等）

**等价的三种写法**:
```tsx
// 写法 1：内联类型（推荐）
export default function Layout({
  children
}: {
  children: React.ReactNode;
}) { }

// 写法 2：定义 interface
interface LayoutProps {
  children: React.ReactNode;
}
export default function Layout({ children }: LayoutProps) { }

// 写法 3：使用 React 内置类型
import { PropsWithChildren } from 'react';
export default function Layout({ children }: PropsWithChildren) { }
```

---

### Q3: 路由分组的名称固定吗？

**回答**: 不固定！可以用任何语义化的名称。

**命名建议**:
- ✅ `(platform)` - 平台
- ✅ `(workspace)` - 工作区
- ✅ `(dashboard)` - 仪表盘
- ✅ `(main)` - 主要功能
- ❌ `(abc)` - 无意义
- ❌ `(temp)` - 不清晰
