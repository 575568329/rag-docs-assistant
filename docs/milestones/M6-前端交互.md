  ```typescript
  async function handleUpload(file: File, kbId: string) {
    const formData = new FormData();
    formData.append('file', file);
    
    const response = await fetch(`/api/kb/${kbId}/upload`, {
      method: 'POST',
      body: formData,
    });
    
    if (response.ok) {
      // 刷新文档列表
      toast.success('上传成功');
    } else {
      toast.error('上传失败');
    }
  }
  ```

- [ ] **拖拽上传**
  ```typescript
  const handleDrop = (e: React.DragEvent) => {
    e.preventDefault();
    const files = Array.from(e.dataTransfer.files);
    files.forEach(file => handleUpload(file, kbId));
  };
  ```

## 💻 核心组件结构

### 对话页
```
ChatPage
├─ Sidebar (对话历史列表)
├─ MessageList
│  ├─ UserMessage
│  └─ AssistantMessage
│     ├─ Markdown 内容
│     ├─ SourceCard[] (来源卡片)
│     └─ EntityTags[] (实体标签)
└─ ChatInput
```

### 数据页
```
DataPage
├─ Sidebar (知识库列表)
└─ KbCard[]
   ├─ KbHeader (名称、文档数、操作按钮)
   ├─ UploadZone (拖拽上传)
   └─ DocumentList
      └─ DocumentItem[] (文件名、操作)
```

### 图谱页
```
GraphPage
├─ Sidebar (收藏节点列表)
├─ ControlPanel (知识库切换、缩放)
├─ GraphSearch (搜索面板)
├─ GraphCanvas (力导图)
└─ NodeDetail (节点详情面板)
```

## ⚠️ 注意事项

1. **文件上传进度**：使用 XMLHttpRequest 或 fetch 的 onProgress
2. **图谱性能**：节点过多时考虑虚拟化或分页
3. **错误提示友好**：不要直接显示技术错误信息
4. **加载状态**：所有异步操作都要有加载提示
5. **权限控制**（可选）：如果是多用户系统，需要权限验证
